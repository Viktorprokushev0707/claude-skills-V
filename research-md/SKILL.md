# SKILL: Оптимизированный веб-поиск

## Когда применяется
Пользователь хочет собрать информацию из интернета на любую тему:
- "исследуй X"
- "найди информацию о Y"
- "собери всё что известно про Z"
- "сравни A и B"
- "что нового в X за последнее время"

---

## Шаг 0: Уточни формат результата (ОБЯЗАТЕЛЬНО, до любых поисков)

Прежде чем искать — спроси пользователя:

```
Прежде чем начать, уточни:

1. **В каком формате нужен результат?**
   - Краткое резюме (1-2 страницы с главным)
   - Структурированный профиль (таблицы + разделы)
   - Сравнительная таблица (если несколько объектов)
   - Хронология событий (что и когда произошло)
   - Полный дамп (только сырые данные, без обработки)
   - Свой формат: ___

2. **На какую глубину копать?**
   - Обзор (5-7 страниц, ~10 мин)
   - Среднее (8-12 страниц, ~20 мин)
   - Глубокое (15+ страниц, ~40 мин, 500 лимит markdown.new)

3. **Есть ли приоритетные источники?**
   - Только официальные (сайты компаний, документация)
   - Новостные и аналитические тоже
   - Всё подряд
```

Жди ответа. Только после этого переходи к Шагу 1.

---

## Шаг 1: Найди источники (WebSearch, 3-5 запросов)

На основе темы и уточнённых параметров — составь список URL для скачивания.

**Правила отбора:**
- Ищи конкретные страницы, не главные страницы сайтов
- Приоритет: официальная документация → блоги → аналитика → новости
- Не бери: вакансии, страницы о команде, условия использования
- Для каждого URL запиши зачем он нужен (что там должно быть)

Составь список в формате:
```
1. https://... — [что там и зачем]
2. https://... — [что там и зачем]
...
```

Покажи список пользователю и скажи: "Начинаю скачивать X страниц"

---

## Шаг 2: Скачай контент через markdown.new

Для каждого URL:

```bash
curl -s -X POST https://markdown.new/ \
  -H "Content-Type: application/json" \
  -d '{"url": "URL_СЮДА"}' | python3 -c "
import json, sys
d = json.load(sys.stdin)
if d.get('success'):
    print('=== SOURCE:', d.get('title', ''))
    print('=== URL:', d.get('url', ''))
    print(d['content'][:5000])
    print('--- END ---')
else:
    print('FAILED')
"
```

**Лимиты и правила:**
- Максимум 15 страниц за сеанс (из 500/день лимита markdown.new)
- Для длинных страниц берём первые 5000 символов (последние/самые важные данные обычно вверху)
- Если страница вернула FAILED — пропускаем, отмечаем в отчёте

**Во время скачивания веди учёт:**
```
✓ https://... (4 230 симв)
✓ https://... (7 810 симв)
✗ https://... — FAILED
```

## Шаг 3: Сохрани сырые данные

Создай папку: `web_research/results/{тема}_{дата}/`

Файл 1 — **всё скачанное** (сырой дамп):
```
web_research/results/{тема}_{дата}/raw.md
```

Формат файла:
```markdown
# Сырые данные: {тема}
Собрано: {дата и время}
Страниц скачано: N из M

---

## Источник 1: {title}
URL: https://...

{полный скачанный контент}

---

## Источник 2: {title}
...
```

---

## Шаг 4: Создай результат в запрошенном формате

Файл 2 — **финальный результат**:
```
web_research/results/{тема}_{дата}/result.md
```

Наполни согласно тому формату, который выбрал пользователь на Шаге 0.

В начале файла всегда добавляй шапку:
```markdown
# {Тема} — {Формат результата}

**Дата исследования:** {дата}
**Источников проверено:** N
**Страниц скачано:** M (через markdown.new)
**Не открылось:** K

---
```

В конце файла всегда добавляй список источников:
```markdown
---
## Источники
- [Название](URL) — что взяли отсюда
- ...
```

---

## Шаг 5: Отчёт пользователю

```
## Готово: {тема}

📁 Файлы сохранены в: web_research/results/{тема}_{дата}/
   ├── raw.md      — всё скачанное (X симв, Y страниц)
   └── result.md   — {запрошенный формат}

**Скачано страниц:** M из N найденных
**Не открылось:** K (перечисли какие)

### Главные находки:
[3-5 самых важных вещей, которые нашёл]
```

---

## Быстрый старт (шпаргалка)

```
0. Спросить формат → 1. WebSearch (3-5 запросов) → 2. curl markdown.new (до 15 URL)
→ 3. Сохранить raw.md → 4. Написать result.md → 5. Отчёт
```

## Ограничения (честно)
- markdown.new: 500 запросов/день на IP, только публичные страницы
- JS-сайты с авторизацией (LinkedIn, Twitter) — не работает
- Очень длинные страницы берём частично (первые 5000 симв)
- Платный контент (Bloomberg, The Information) — не доступен
